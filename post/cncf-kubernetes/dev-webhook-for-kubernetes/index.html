什么是kubernetes webhook

痛点

网上很多文章都是说写了kubernetes webhook代码后编译丢到k8s里面去测试,但是这样就非常消耗时间和精力,因为无法立刻知道写的代码是否有问题.

加多一层

IT界有句老话,所有的问题都可以通过加多一层来解决。

这句话成为了本篇文章的核心所在。

怎么加多一层从而方便的进行webhook开发呢?
