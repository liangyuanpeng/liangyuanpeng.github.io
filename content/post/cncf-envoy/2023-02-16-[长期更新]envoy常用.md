# 场景以及配置

## 权重分流

基于权重做灰度发布.

```yaml
- match:
    prefix: /
  route:
    weighted_clusters:
      clusters: 
        - name: lank8scn_v1
          weight: 50
        - name: lank8scn_v2
          weight: 50
      total_weight: 100
```

上述配置部分表示匹配前缀为 `/` 的请求并且按照 `1:1` 的权重分流给 `lank8scn_v1` 和 `lank8scn_v2`.

## header分流

基于 header 做灰度发布.

```yaml
                        - match:
                            prefix: "/"
                            headers:
                            - name: sversion
                              string_match:
                                exact: "v2"
                          route:
                            cluster: lank8scn_v2
                        - match:
                            prefix: "/"
                          route:
                            cluster: lank8scn_v1
```

上述配置部分表示匹配前缀为 `/` 的请求并且将包含有  header 为 `sversion:v2` 的请求分流给`lank8scn_v2`,而没有这个 header 时将请求正常分流给 `lank8scn_v1`.
